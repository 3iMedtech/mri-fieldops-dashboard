## 5. Production pre-flight read-only queries

Run only after the appropriate Gate A/B phrase for that track.

### 5.1 Common pre-flight (all tracks; read-only)

```sql
-- A: helpers present
select p.proname, p.prosecdef, p.proconfig
  from pg_proc p join pg_namespace n on n.oid = p.pronamespace
 where n.nspname = 'public'
   and p.proname in (
     'app_user_role','app_can_write','app_is_admin',
     '_other_active_admin_exists',
     '_user_roles_block_last_admin_delete'
   )
 order by p.proname;
-- expected: 5 rows; prosecdef=true; proconfig contains search_path=public
```

```sql
-- B: lifecycle tables present
select c.relname
  from pg_class c join pg_namespace n on n.oid = c.relnamespace
 where n.nspname = 'public'
   and c.relname in ('config_assets','asset_lifecycle','asset_lifecycle_history','cmc_contracts','pm_schedule','user_roles')
 order by c.relname;
-- expected: 6 rows
```

```sql
-- C: 0006 NOT yet present (Track A pre-flight)
select count(*) as renew_rpc_present
  from pg_proc p join pg_namespace n on n.oid = p.pronamespace
 where n.nspname = 'public' and p.proname = 'app_renew_asset';
-- expected: 0
```

```sql
-- D: 0007 NOT yet present (Track B pre-flight)
select count(*) as cmc_sn_unique_present
  from pg_constraint
 where conrelid = 'public.cmc_contracts'::regclass
   and conname  = 'cmc_contracts_sn_unique';
-- expected: 0
```

```sql
-- E: row count baseline
select 'config_assets' as t, count(*) from public.config_assets
union all select 'cmc_contracts', count(*) from public.cmc_contracts
union all select 'asset_lifecycle', count(*) from public.asset_lifecycle
union all select 'user_roles', count(*) from public.user_roles;
-- expected: config_assets=25 (or current), cmc_contracts=13 (or current),
--           asset_lifecycle=0 (Phase 2 production baseline),
--           user_roles=3.
```

## 9. Tag gate (unified v1.4.2)

Run only after **all in-scope tracks** have passed production runtime smoke. Per `L-RPM-006`:

1. Bump `VERSION` → `1.4.2`.
2. Update `index.html` version metadata (4 constants).
3. Add `## [1.4.2] — <UTC date>` entry to `CHANGELOG.md`.
4. Run `./scripts/release.sh 1.4.2` to produce `releases/v1.4.2/MANIFEST.txt` + `releases/v1.4.2/index.html`.
5. Verify CHANGELOG date matches MANIFEST `date:` field per `L-RPM-006` UTC-date pitfall guidance.
6. Commit "release: v1.4.2"; push main; observe Pages workflow.
7. After Pages workflow success, verify `curl -s prod/index.html | shasum -a 256 == MANIFEST.sha256` byte-equality (canonical `L-RPM-006` acceptance check).
8. Create annotated tag: `git tag -a v1.4.2 -m "v1.4.2"`; push tag.
9. Post-tag doc-memory commit (separate phrase: `commit STATE.md + memory updates after v1.4.2 tag`).
