## 15. PASS / HOLD / STOP Criteria

The Delivery Orchestrator (or Database PM for SQL-only stops; Runtime PM for runtime-only stops; Release PM for deploy/tag stops) aggregates findings at each stop point.

### 15.1 PASS

Every check in the current stop-point's expected-output column matches. Advance to the next stop-point's pre-requisites. Wait for the next operator phrase if the next stop is gated.

### 15.2 HOLD

One or more checks are missing data (paste-back incomplete) or show ambiguous results that require operator clarification. Do **not** advance. Do **not** rollback (no failure yet). Wait for operator input.

### 15.3 STOP

One or more checks **failed** (output does not match expected). Do **not** advance. Reply `stop` in chat. Consider rollback (§13). Operator may issue Gate G to authorize the rollback flow. **No forward action proceeds until either rollback completes or the operator explicitly resolves the discrepancy.**

### 15.4 ESCALATE

A finding cannot be classified by this runbook (e.g., the production migration ran but a downstream service the runbook doesn't enumerate is reporting errors). Stop, summarize the unknown, and ask the operator for direction. Specialists (sql-rls-safety, runbook-verifier, runtime-integration, release-pm) may be brought in for the unknown.

### 15.5 Gate-level rollups

## 17. Risks (production-specific)

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Production V2 missing code is something other than `AN025` (e.g., 0 codes, or 2+ codes including a non-AN025) | Low | Medium | §5.4 E + E2 catch this pre-apply. STOP and reconcile. |
| GRANT INSERT/UPDATE for `authenticated` not present on one of the 3 tables | Low | High | §5.1 G catches pre-apply (`L-SQL-004`). Without GRANT, RLS evaluation is a no-op and Manager writes silently fail with `permission denied` instead of expected RLS denial. |
| Helper functions missing or non-SECURITY-DEFINER on production | Low | High | §5.1 A catches. STOP and reconcile by re-running Phase 1 helper setup. |
| user_roles seed missing or wrong on production | Low | High | §5.1 B catches. Without correct seed, Manager resolves to `viewer` even with new policies. |
| 0005 apply rolls back due to invariant failure | Very low | Medium | Transaction-wrapped; no partial commit possible. SQL Editor shows the ERROR. §13.3 + §10 unchanged. |
| Pages deploy fails after merge (rare; Actions runner glitch) | Low | Medium | Re-run `pages-deploy.yml` via `gh workflow run` or Actions UI. If repeated failure, escalate to release-pm. |
| Realtime CHANNEL_ERROR on production (staging analog noted; non-blocking) | Medium | Low | Single-user UX unaffected. Track for v1.4.2 polish. Do not block release. |
| Manager XLSX gate accidentally relaxed (`L-RTI-008`) | Very low | High | §12.4 UI smoke explicitly checks Manager cannot see XLSX upload. If visible to Manager, STOP — this is a regression of the function-level Admin gate. |
| Non-admin role escalates to admin under RPC failure (`L-RTI-007`) | Very low | CRITICAL | §12.5 explicitly tests this. If observed, IMMEDIATE STOP + Gate G + Gate H. Notify operator. |
| Merge commit on main contains protected-file changes the runbook didn't authorize | Low | High | §11.2 git diff catches pre-merge. STOP if non-empty. |
| `AN026` / `AN027` (staging fixtures) accidentally inserted into production | Very low | Medium | Staging fixtures live only in staging `config_assets`. The 0005 migration only inserts V2 codes (AN001..AN025). Manual SQL would be needed to introduce them — out of scope for this runbook. |
| Stale STATE.md cross-environment fact causes mis-routing | Low | Medium | `L-AM-002`: every cross-env fact in STATE.md has been re-verified via §5 pre-flight before any apply. |
